## 1. 개요

### 1.1 주제 (한 문장)

4×4 부분 마방진(빈칸 2개, 합 34)을 다룰 때, 조합·노가다·수동 검산에서 생기는 **시간 낭비**와 **“맞는지 모르겠다”는 불안**을 줄이기 위해, 풀이 과정과 결과를 **반복 검증할 수 있는 계약(INV-*)** 을 먼저 발견한다.

### 1.2 표면 문제 (잘못된 정의 — 하지 않을 것)

| 표면 문제 | 이번 프로젝트에서 **하지 않음** |
|-----------|--------------------------------|
| 「4×4 부분 마방진 **ECB 솔버 프로그램/UI**를 만든다」 | GridUI · InputHandler · ResultDisplay 등 **Boundary UI 구현** |
| 「**더 큰 N×N** 마방진도 푼다」 | 5×5 이상 확장 · 일반 생성 알고리즘 |
| 「**힌트·자동완성·공유 화면**을 넣는다」 | Mom Test에서 나오지 않은 **기능 가설** |
| 「**모든 마방진 생성법**을 가르친다」 | 교재·백과식 알고리즘 전범위 |
| 「실패 조건 3개를 **이름만** 나열한다」 | 테스트·불변식 **없이** 문서만 작성 |

### 1.3 진짜 문제 (한 문장)

마방진을 손으로 풀 때 빈칸·숫자 조합을 **경우의 수처럼** 맞춰야 해서 시간과 노력이 커지고, 답을 찾아도 **‘노가다로 맞춘 것’** 이라 정답과 풀이에 대한 **확신**이 생기지 않는다.

## 2. Mom Test

### 2.1 페르소나

4×4 격자, **빈칸 2개(0)**, **1~16**, **행·열·대각선 합 34**를 손·코드로 다루는 **학습자**.

### 2.2 증거 (인터뷰 인용)

1. 「그걸 모르고 모든 조합을 맞춰보는 게 너무 오래 걸리고 힘들게 느껴졌어. 모든 가능한 경우의 수를 다 대입해 보는 것 같았어.」
2. 「시간이 오래 걸렸지만 지루하진 않았어. 자신감도 들지 않았어 왜냐면 노가다로 해결했으니까.」
3. 「원리를 알고 하니까 훨씬 빨랐어.」 / 「이제는 내가 마방진을 알았다는 생각에 기뻤어.」

### 2.3 Mom Test 채점 (7/10) — 보완 필요

| 체크 | 상태 |
|------|------|
| 미래 가정 없음 | 증거·진짜 문제 ✅ / 워크북 SC는 세션 목표(미래) ⚠️ |
| 과거 행동·시간·실수 구체성 | 행동 ✅ / **몇 분·틀림 순간** 약함 |
| 진짜 문제에 솔루션名 없음 | ✅ |
| 표면 ↔ 진짜 분리 | ✅ |
| 도메인(4×4, 10선, 34, 빈칸 2) | 페르소나만 ✅ / 증거는 완성·확대 마방jin 중심 ⚠️ |

**보완 질문 (1개):**  
「가장 최근에 **14칸은 채워져 있고 빈칸이 2개(0)인** 4×4 격자를 풀 때, **그 두 빈칸에 넣을 숫자**를 찾기 위해 **종이·손으로 실제로 한 첫 행동**이 뭐였어?」

## 3. 도메인

### 3.1 마방진 정의

- **크기:** 4×4
- **숫자:** 1~16, 각각 **한 번씩**만 사용
- **10선:** 4행 + 4열 + 2대각선 — 각 합 = **34**
- **부분 문제:** 14칸은 주어지고, **2칸은 0(빈칸)** → 빈칸 위치·값을 찾아 완성

### 3.2 예시 입력 (슬라이드)

```
16   3   2  13
 5  10  11   0    ← 빈칸
 9   6   0  12    ← 빈칸
 4  15  14   1
```

목표: 빈칸 2곳에 남은 숫자를 넣어 **10선 합 34** · **1~16 중복 없음**을 만족.

### 3.3 ECB (개념 분류 — 구현 범위는 §5 참고)

| 계층 | 후보 클래스 | 역할 |
|------|-------------|------|
| **Entity** | `MagicSquare`, `Cell`, `SolveResult` | 4×4 격자 데이터·상태·풀이 결과 |
| **Control** | `SquareValidator`, `MissingFinder`, `Solver` | 합 검증 · 빈칸 탐색 · 완성 |
| **Boundary** | `GridUI`, `InputHandler`, `ResultDisplay` | 시각화 · 입력 · 결과 표시 (**이번 범위 밖**) |

## 4. R-G-I-O

| | 내용 |
|---|---|
| **Role** | 4×4 부분 마방진을 종이·코드로 풀고, ECB로 역할을 나누려는 **학습자** |
| **Goal** | 빈칸 2곳에 1~16 중 남은 숫자를 넣어, **10선 합 34** · **1~16 한 번씩** 을 **스스로 확신**할 수 있다 |
| **Input** | 4×4 정수 격자. 14칸은 1~16, **2칸은 0(빈칸)** |
| **Output** | ① 빈칸 2곳의 **위치·값** ② **판정** — 유효 / 무효 / 판정 불가 ③ 무효 시 **깨진 조건** (행·열·대각선·중복·범위 등) |
